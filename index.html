3D打印和激光切割保姆级教程

3D打印教学

本教程在黎明杯培训期间编写，帮助参赛队员们使用3D打印和激光切割加工自己的零件。也欢迎分享给其它同学。

面向小白的上手教程，需要进阶技巧的同学请查阅相关资料。

送加工之前需要设计建模。本教程中建模软件都为SolidWorks。

生成STL文件

打开绘制的3D图纸，文件，另存为

选择STL格式

保存

学创3D打印机远程打印（推荐）

点击进入学创3D打印云平台

首次使用需填写联系方式

进入个人中心

点击修改个人信息

填好保存

上传并打印

点击上传3D文件

选择并打开要打印的零件的STL文件

可见上传成功。材料就用PLA, 实心空心不用管（这里调了也没用hhh）

点击右下角立即申请

填写自己的学院班级，指导老师填哪位都影响不大

打印用途简单填写即可，在这里可以备注想要的材料颜色和打印密度需求

例如黎明杯机械赛 颜色最好黑色 填充密度85% 辛苦助管力

点击提交订单

然后进入待审核流程，审核通过就会开始打印

打印好之后会有邮件通知，没收到通知可能会在垃圾邮件里

线下取件在3D打印中心，地点在学创C楼，进门右转C101。建议晚上十点之前去取。

龙宾楼3D打印机打印

安装FlashPrint软件

首先进入官网下载中心

下载对应版本的FlashPrint

解压下载的压缩包，运行exe

windows可以以管理员身份运行，安装成功率会高一些

安装成功

导入STL文件并切片

打开FlashPrint软件

界面如图

基础设置

选择机器类型，唐创打印机是 Guider II S

选择喷嘴尺寸，唐创打印机选 0.4 mm

导入STL文件并调整

载入文件

选择要打印的STL并打开

导入后，角度和位置都是默认还不错的。图上模型在正方体内的位置即是打印时模型在打印机里的位置

点击右边这一列可以调整视角、方向、位置等

例如旋转，可以左键点住环进行三个自由度上的旋转

可以每90度进行翻转

可以按面放平（勾选之后点击要放到底面的面）

切片并保存

点击开始切片

喷嘴尺寸选择PLA，切片配置可选

对3D打印了解比较多的同学可以进入专家模式进行打印设置

大家对于填充密度的修改需求可能会比较多，即使是精细模式，15%的填充密度对于结构件来说还是太小了

可以修改填充密度，改到85%就算很大了，不需要100%。

调整完毕后点击切片。

切片完成之后点击保存到本地

会保存成一个.gx文件

丢进打印机打印

打印机如果没有开，可以自己打开。船型开关在打印机的背面。打印机的触摸屏的UI还是比较符合直觉的，就不放图片了。触摸屏质量不大好，点击需要有耐心。

  1. 把.gx文件存进U盘
  2. U盘插入打印机（插口在右侧）
  3. 在打印机的屏幕上打开U盘
  4. 选择要打印的文件
  5. 打印

激光切割教学

生成 DXF 图纸

首先需要完成设计（如小车底盘，机械臂等）

注意注意：激光切割会有加工误差，概括为“内缩外扩”。也就是说切割的边缘会多切掉 0.5~1mm 。因此设计尺寸的时候记得稍作补偿，尤其是螺丝孔等较精细的地方。

选中零件，右键，导出为dxf

点勾确认

查看并保存。应该是二维图纸

将保存的文件存入U盘，带去学创切割

去学创切割

激光切割中心在学生创新中心C楼，进门，左转，直走，在最靠里的位置。

交我办预约

建议到场再约也不迟。

交我办->场馆预约->激光切割->预约对应时段

填写材料加工申请单

现场领取，正常填写即可。指导老师的填写合适即可，联系可以不写。

开切

U盘文件交给助管即可。虽然自己操作不算难，但建议按流程按规定，交给助管吧。亚克力或木板切割可能会有起火危险，不要慌乱，伟大的助管会妥善灭火处理。

材料一般来说切割中心有板材，如果没有的话，需要到设备图书馆申领。学创设备图书馆

EdgeTX系统基础（含inav脚本刷写）

本博客是对EdgeTX系统基础操作的一个整理记录，基于RadioMaster Pocket，黑白屏幕

主要按照遥控器拿到手的一系列配置顺序撰写

点击进入官方文档

系统设置

校准摇杆

  1. SYS （按动SYS系统按键）
  2. PGDW -> 6 （点击PAGE>至第六页）
  3. Calibration （按动滚轮确认）（UI显示按动ENTER的地方即是按动滚轮）
  4. 根据提示完成校准，打杆要拉满行程
  5. RTN （按动RTN返回按键）

ELRS设置

进入

  1. SYS
  2. ExpressLRS ENTER， 进入ELRS的LUA脚本文件
  3. 等待刷新完成

调整基础参数

  • Packet Rate 刷新率（包转发速率） 250Hz差不多
  • Telem Ratio 回传比例 比例越低，遥测频率和遥测数据流越大（比值的基准是打包速率，如1：2就是每打两个包等一个回传数据）
  • Switch Mode 选择的是辅助通道打包策略 这里选Hybrid 参考ELRS文档
  • TX Power 发射功率 不用有功率焦虑，室内25mw，户外100mw够了，也可开满内置的250mw（非远航）

更多功能

  • Model Match 我也没有很多飞机在飞。Off关掉就好

  • VTX Administration 直接在遥控器里更改图传选项 须在Betaflight里的VTX table做相关配置

    • Band 频点设置
    • Channel 通道选择
    • Power level 功率选择
    • 点击 Send 发送文件进行配置

  • WiFi Connectivity

    • 高频头wifi的打开 可以电脑或手机连接wifi打开ELRS网页端地面站（修改对频密码等）
    • 也可以打开接收机的wifi
    • backpack不怎么用，不管

  • BLE Joystick 蓝牙连接电脑（用来玩模拟器等）

  • Bind 对频配对

    • 飞机通电三次保持，点击bind对频（未设置对频密码）

PAGE 3 RADIO SETUP

  • 设置日期、时间、声音等
  • 选择AETR1234的经典通道设置
  • 选择Mode2 美国手

模型设置

PAGE 1 MODELSEL

选择模型

PAGE 2 SETUP

要在这里打开内置高频头，看到红灯亮起即打开（针对RM Pocket）

PAGE 6 MIXES

混控输出

  • 前四个通道就是四个遥感
  • 新建的模型CH5及往后都是空的，需要手动映射。配置即将遥控器上的按键和CH5,CH6等对应起来
  • 选中某一个通道，点击Edit进行配置。其实主要把Source配置好就行，名字也可以简单命名一下
  • 遥控里将按键与通道对应，飞控再将通道与功能对应，即可实现使用遥控器按键、开关、拨轮等控制飞机功能

inav脚本文件设置

  1. 仓库地址：https://github.com/iNavFlight/OpenTX-Telemetry-Widget
  2. 下拉，在Setup里点击 Download latest release

  3. 下载后解压文件夹，打开

  4. 把这两个子文件夹之间拷贝到遥控器的SD卡里，同名文件夹替换即可

    (这个自带的512Mb的U盘就是很慢)
  5. 把卡插回遥控器。连接上飞机。长按TELE，即可打开遥测回传界面。看到电压、GPS等信息

柱状图

  • 双向柱状图
  • 调柱子宽度
  • 调颜色
  • 数值标签
import matplotlib.pyplot as plt

# 初始化一个图形和轴对象
fig, ax = plt.subplots(figsize=(10, 7))  # figsize=(宽度, 高度)，单位为英寸，指定图形的大小

# 有5个比赛集
set_numbers = [1, 2, 3, 4, 5]  # 每个集的编号
width = 0.45  # 每个柱子的宽度

# 每位选手在每个集中赢得的游戏数
p1_games = [1, 7, 6, 3, 6]  # 玩家1赢得的游戏数
p2_games = [-6, -6, -1, -6, -4]  # 玩家2赢得的游戏数，使用负数以在x轴下方显示

# 绘制柱状图
rects1 = ax.bar(set_numbers, p1_games, width, label='Alcaraz', color='#a4e66b')  # 绘制玩家1的柱状图
rects2 = ax.bar(set_numbers, p2_games, width, label='Djokovic', color='#3fb1e3')  # 绘制玩家2的柱状图

# 设置y轴的范围
ax.set_ylim(-8, 9)
# 设置x轴和y轴的标签
ax.set_xlabel('Set Number', fontsize=23)  # 设置x轴的标签及其字体大小
ax.set_ylabel('Number of Won Games', fontsize=23)  # 设置y轴的标签及其字体大小
# 设置图表的标题
ax.set_title('Number of Won Games per Set', fontsize=23, pad=15)  # 设置标题及其字体大小和上边距

# 设置x轴的刻度
ax.set_xticks(set_numbers)  # 设置x轴上的刻度位置，与比赛集编号对应
# 添加图例
ax.legend(loc='upper left', bbox_to_anchor=(0, 1), fontsize=19)  # 设置图例的位置和字体大小

# 绘制x轴上的0线
ax.axhline(0, color='black', linewidth=0.8)  # 绘制x轴（y=0）的线
# 隐藏y轴的刻度标签
ax.set_yticklabels([])  # 将y轴上的刻度标签设置为空列表，以隐藏它们
# 设置x轴刻度标签的大小
ax.tick_params(axis='x', labelsize=23)  # 设置x轴上的刻度标签的字体大小

# 在每个柱子上方或下方添加数值标签
for rect in rects1 + rects2:
    height = rect.get_height()  # 获取柱子的高度
    # 根据柱子的位置，添加相应的数值标签
    ax.text(rect.get_x() + rect.get_width() / 2., 1.01*height, '%d' % int(abs(height)),
            ha='center', va='bottom' if height > 0 else 'top', color='black', fontsize=23)

fig.tight_layout()  # 调整子图参数，使之填充整个图形区域
plt.show()  # 显示图形

ELRS固件刷写

ELRS的接收机和发射机都需要刷写好固件，并且固件大版本号匹配才能对频配对。本博客以ELRS 2.4G 接收机刷写为例，介绍一种成功率较高的刷写方式。即：

  • 官方仓库下载源码（简单且靠谱且新）
  • 使用ELRS配置器本地编译固件（软件下载容易本地编译稳定）
  • 通过wifi连接，使用网页端上传和烧录固件（不会因为端口问题出现各种bug）

下载固件源码

  1. 进入ExpressLRS仓库
  2. 克隆仓库或直接下载zip，zip需解压备用

编译固件

安装 ExpressLRS configurator

官网安装教程

官方Releases页面，Assets中选择对应的安装文件

安装完成并打开软件

编译固件

我们需要通过刚安装的ELRS配置器将固件的源代码编译成可以烧录到硬件上的二进制文件(.bin)

选择本地

本地路径需选择到源码的src路径

先选设备分类后选设备。若不清楚应咨询硬件购买店铺的客服

选择WIFI

设备选项基本可以默认。

  • 监管区域不要选欧洲的限制会多。
  • 对频短语设置建议勾选并自己设置（发射机需要设置相同的对频密码），这样就不需要使用插拔断电三次这种不优雅的方式进行对频了
  • AUTO_WIFI_ON_INTERVAL的值可以自己调，这里填60就是插电后60秒内未对频将进入wifi模式

点击编译，编译，编译，不要直接刷写

编译中，这一步时间较久，我这边用了2分钟

编译成功，弹出编译好的固件所在的文件夹，名称类似于箭头所指的是我们要的固件

上传并烧录固件（通过wifi，通过网页端）

wifi连接

通电无遥控器连接，AUTO_WIFI_ON_INTERVAL超时后，接收机自动WiFi upade模式（指示灯闪烁情况会变化）。

连接 ExpressLRS RX 的 wifi, wifi密码为expresslrs

自动弹出网页页面。

这里选择文件选择刚才编译出的bin文件,点击Update

uploaded 100% 后不着急，还需要flash

显示烧录成功。官方说要等待到LED blinking, 我也不清楚这里的blinking是快闪还是慢闪。Anyway，等一小会看到led闪烁模式发生变化，就好了。

在Clash for Windows中复制终端代理命令

问题背景

运行Python网页自动化脚本失败，在终端中向谷歌发送测试请求失败，原因应该是被墙了。虽然已经用clash开了代理但是没有用，需要在终端输入命令配置代理

解决方法

打开Clash for Windows, 选择General标签

点击Port里的终端小图标

点击Copy Commands Only

点击Power Shell, 命令已经被复制到剪贴板了

回到PyCharm, 粘贴命令，回车运行

再次运行curl http://www.google.com/

出来一堆HTML, 成功

Some Notes

curl 是一个常用的命令行工具，用于传输数据。这条命令是在使用 PowerShell 的 curl 别名来发送一个 HTTP GET 请求到 http://www.google.com。

命令执行的结果显示了对 Google 首页的请求响应。状态码 200 表示请求成功，StatusDescription 显示为 OK，也表明了请求成功。Content 部分显示了返回的 HTML 内容，这是网页的结构和内容。RawContent 包含了 HTTP 响应的头部和正文。Forms、Headers、Images、InputFields 和 Links 是解析后的 HTML 元素，它们分别代表了表单、头部信息、图片、输入字段和链接。

这个命令在多种情况下都可能会用到，例如：

  1. 网页抓取：当你需要从网页上抓取数据时，可以使用这个命令来获取页面内容。
  2. API 测试：如果你在测试一个 HTTP API，你可以使用这个命令来发送请求并查看响应。
  3. 自动化脚本：在自动化脚本中，需要获取外部数据或与其他服务交互，这时可以使用这个命令。

这样的命令可以帮助我们快速地从命令行界面与 Web 服务进行交互。
